# GitHub CLI Multi-Account Token Operations

## Overview

The `gh` CLI uses `~/.config/gh/hosts.yml` for its primary auth state. When automating with a different account or service token, override per-command with `GH_TOKEN` environment variable — no account switching needed, no config file modifications.

## Per-Command Token Override

```bash
# Override for a single command
GH_TOKEN=ghp_xxx gh pr create --title "..." --body "..."

# Or export for a script block
export GH_TOKEN=ghp_xxx
gh pr view 42
gh issue comment 7 --body "done"
unset GH_TOKEN
```

`GH_TOKEN` takes precedence over `~/.config/gh/hosts.yml` for the duration of that command.

## Check Which Account Is Active

```bash
gh auth status
# With override:
GH_TOKEN=ghp_xxx gh api user --jq '.login'
```

## Common Automation Tasks

### Create PR

```bash
GH_TOKEN=$SERVICE_TOKEN gh pr create \
  --title "agent/SH-12 add feature" \
  --body "$(cat <<'EOF'
## Summary
- Implements X
- Adds tests for Y

## Test plan
- [ ] CI passes
- [ ] Manual smoke test

🤖 Generated by Codex
EOF
)" \
  --base main \
  --reviewer ishuru
```

### Request Reviewer on Existing PR

```bash
GH_TOKEN=$SERVICE_TOKEN gh pr edit 42 --add-reviewer ishuru
```

### Post Issue Comment

```bash
GH_TOKEN=$SERVICE_TOKEN gh issue comment 7 \
  --body "PR: https://github.com/org/repo/pull/42"
```

### List Open PRs for a Branch Pattern

```bash
GH_TOKEN=$SERVICE_TOKEN gh pr list \
  --search "head:agent/" \
  --json number,title,headRefName \
  --jq '.[] | "\(.number)\t\(.headRefName)\t\(.title)"'
```

### Force-Push a Branch (for rework)

```bash
git push --force-with-lease origin agent/SH-12-feature-slug
# No GH_TOKEN needed for push — uses git's own auth (SSH key or HTTPS credential)
```

### Merge PR

```bash
GH_TOKEN=$SERVICE_TOKEN gh pr merge 42 --squash --auto
```

## Token Scopes Required

| Operation | Required scope |
|---|---|
| Read repos, PRs, issues | `repo` (or `public_repo` for public repos) |
| Create PRs, comments | `repo` |
| Request reviewers | `repo` |
| Merge PRs | `repo` |
| Push branches | Not a GH_TOKEN scope — handled by git SSH/HTTPS auth |

Generate a token: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) or Fine-grained tokens.

## Fine-Grained Tokens (Preferred)

Fine-grained tokens scope to specific repositories:

1. GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
2. Select repo(s)
3. Repository permissions: `Pull requests: Read and write`, `Issues: Read and write`, `Contents: Read and write`

Classic tokens grant broad access — prefer fine-grained for automation.

## Common Mistakes

| Mistake | Fix |
|---|---|
| `gh` acts as wrong account | Use `GH_TOKEN=...` prefix; don't rely on active `gh auth` login |
| Push fails despite valid GH_TOKEN | Push auth is separate (git SSH key or `GH_TOKEN` only works for HTTPS remotes with `git push`) |
| `gh pr create` fails: no current branch | Must be on the branch; check `git branch` |
| Reviewer not found | Use GitHub login (e.g., `ishuru`), not display name or email |
| Token expires / 401 | Regenerate token; classic tokens can be set to no-expiry but fine-grained expire |
| `GH_TOKEN` vs `GITHUB_TOKEN` | `GH_TOKEN` is the gh CLI env var; `GITHUB_TOKEN` is GitHub Actions auto-token |
